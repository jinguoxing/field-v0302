
> 约定：
>
> * 所有组件都走受控表单（value/onChange）
> * `bind` 字段在生成器层负责把 props.value 绑定到 formState 路径
> * Schema 用 Draft 2020-12 风格（可简化实现）

# 3) 最小建议：生成器如何用这些 props + schema

* 每个 Tab 的 `domain` 对应一个 schema（上面 2.x）
* 前端保存时：

  1. 先用 schema 校验本地 payload
  2. 再 `PUT /sections/{domain}` 保存
* 后端保存时：

  1. 用同一 schema 二次校验
  2. 按 domain 分表/分 JSON 存储（你们之前讨论过“哪些 YAML 入配置表”）
